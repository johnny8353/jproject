1 表格制作

1.1 查询表格

查询数据时使用查询表格样式。

注意：

  1. table样式为：class="tb_input" 且 cellspacing="1"
  2. 表头td样式为：class="search"
  3. 表格一般分为4例，标题td样式为：class="td_title",标题列占百分比为10%-15%

图例：

代码：

  <table cellspacing="1" cellpadding="0" class="tb_search">
    <tr>
      <td colspan="4" class="search">请填写查询条件</td>
    </tr>
    <tr>
      <td class="td_title" width="12%">单据号</td>
      <td width="40%"><input type="text" name="txtNo" class="txt_120"/></td>
      <td class="td_title" width="12%">申请人</td>
      <td><input type="text" name="txtPerson" class="txt_120"/></td>
    </tr>
    <tr>
      <td colspan="4" align="right">
        <input name="btnSearch" type="button" value="查询" class="button" onclick="searchresult();"/>
        <input name="btnReset" type="reset" value="清空" class="button"/>
      </td>
    </tr>
  </table>

1.2 信息表格

新增、查看信息时使用信息表格样式。

注意：

  1. table样式为：class="tb_input" 且 cellspacing="1"
  2. 表格一般分为4例，标题td样式为：class="td_title",标题列占百分比为10%-15%

2 控件使用

2.1 日期控件

先在"head"之间引入日期控件JS,然后把日期控件代码拷贝过来即可

注意：

  1. 引入日期控件时JS时，路径要正确
  2. img的onclick事件中的参数一定要与input的name相同
  3. 在同一页面存在多个日期控件时，各日期控件的name要保持唯一性

图例：

代码：

     <script type="text/javascript" src="../../js/dateTextBox.js"></script>
     <input name="dateStart" class="txt_120" onChange="fCheckDateTime('dateStart')"/>
     <img border="0" src="../../../images/icons/calendar.gif" width="16" height="15"  style="cursor:pointer"
          onclick="javascript:fPopCalendar(dateStart,dateStart,dateStart.value );"/> 至
     <input name="dateEnd" class="txt_120" onChange="fCheckDateTime('dateEnd')"/>
     <img border="0" src="../../../images/icons/calendar.gif" width="16" height="15"  style="cursor:pointer"
          onclick="javascript:fPopCalendar(dateEnd,dateEnd,dateEnd.value );"/>

2.2 翻页控件

翻页控件为table的最后一行，放在"tbody"外或“tfoot”中

注意：

  1. tr样式为：class="tr_pagenumber"
  2. 翻页控件td的colspan参数值应等于数据行列值，否则在IE8下会出错。如：数据行有7列，则colspan="7"

图例：

代码：

     <tr class="tr_pagenumber">
        <td colspan="7"> 第1页/共1页  共5条记录  每页
          <select name="select" >
            <option selected>10</option>
            <option >20</option>
            <option>20</option>

            <option>全部</option>

          <img src="../../../images/icons/nextpage.gif" title="下一页" class="paginate"/>
          <img src="../../../images/icons/lastpage.gif" title="末页" class="paginate"/></td>
      </tr>

2.3 排序控件

对查询结果进行排序

注意：

  1. 引入日期控件时JS时，路径要正确
  2. jquery.js要放在引用JS文件的最新面
  3. js代码中的"#myTable"为排序table的id
  4. 标题tr必须放在"thead"之间
  5. 标题列标签使用th而非td
  6. 点击查看更多关于排序控件的使用帮助

    <script type="text/javascript" src="../../js/jquery.js"></script>

	 $(function() {

            4:{sorter: false}, 5:{sorter: false}, 6:{sorter: false}}});

6 常用js

6.1 页面跳转

页面跳转可用超链接或js实现，它们的效果是一样的。

代码：

     超链接：<a href="aa.html">超链接</a>
    js实现：<a href="#" onclick="location.href='aa.html'">js实现</a>
    操作按钮:<input type="button" class="button" value="新增" onclick="location.href='aa.html'"/>
    返回、取消、上一步：onclick="history.go(-1)",其中参数“-1”为回退的次数，如“-1”表示回退一次

6.2 弹出窗口

点击链接或按钮弹出模态窗口，在规范demo中，只需要引入jquery.js、main.js,然后使用函数：onclick="openwin('aa.html','',650,400)",aa.html为弹出窗口的页面，650为弹出窗口的长度，400为弹出窗口高度，弹出窗口在界面上居中显示。

代码：

   通过链接弹出窗口：<a href="#" onclick="openwin('aa.html','',650,400)"></a>
   通过按钮弹出窗口:<input type="button" class="button" value="新增" onclick="openwin('aa.html','',650,400)"/>

6.3 显示/隐藏

显示/隐藏可通过两种方式实现：

  1. display隐藏的对象不保留其占据的物理空间: style="display:none"为隐藏，style="display:"为显示
  2. visibility隐藏的对象保留其占据的物理空间: style="visibility:hidden"为隐藏，style="visibility:visible"为显示

代码：

   假设有一块区域或元素的id="test"，以下两种方式都可实现显示隐藏：
   <script type="text/javascript">
	   function show()
	   {
		document.getElementById("test").style.display = "" //display显示
	   }
	   
	   function hide()
	   {
		document.getElementById("test").style.display = "none" //display隐藏
	   }
	   
	   //注意：使用visibility隐藏时，仍会占据空间
	   function showV()
	   {
		document.getElementById("test").style.visibility = "visible" //visibility显示
	   }
	   
	   function hideV()
	   {
		document.getElementById("test").style.visibility = "hidden" //visibility隐藏
	   }
   </script>


7 FAQ

列举在制作demo过程中，常见的问题及解决办法

  1. 在原有demo基础上加入新规范demo中的效果或新老demo合并，应该怎么做？
    第一步 文件拷贝：把新规范demo中的js文件、css文件分别拷入原demo中的js、css文件夹内(不会冲突)，把新规范demo的images文件夹内的icons和login文件夹拷贝到原demo的images文件夹内;
    第二步 引用文件：对于新页面，可直接把新规范中的页面拷贝过来，然后修改js、css的路径即可；修改老页面，则要先要保证页面的"!DOCTYPE"及"html"是规范的，一些小问题都是由于它们不规范引出的： 
    <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
"http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
然后引入js及main.css，注意要把main.css放在cssCn.css后面（即让main.css覆盖cssCn.css）
  2. 规范demo中提供的main.css样式不能满足需求，应该怎么办？
    出现这种情况时，可以在“common.css”内新增样式，或通知UI添加新样式，严禁在main.css内新增样式
  3. 系统demo上传到svn后，直接预览览出错，应该怎么解决？
    解决办法请参照文档：Demo 预览出错解决办法.doc
