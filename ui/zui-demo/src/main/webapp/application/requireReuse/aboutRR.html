1. 什么是需求复用:
    需求复用指采用可复用的组件完成需求开发。可复用的组件包括系统用例、界面原型等，其它类似领域的需求文档，或者来自其他人的经验。
  2. 需求复用的好处：
    更稳定的功能，更快的交付，更一致的用户使用体验。
  3. 需求复用方法：
    1. 针对CRUD和查询开发了7种常用的、通用的、可修改的Demo和用例模板，后续会继续补充。
      • CRUD－直接修改方式，适用实体属性较少的场景。
      • CRUD－弹出页修改方式，适用于实体属性较多的场景。
      • CRUD－主从单据方式，适用于单据类，一个主体多个附属内容的场景。
      • Query－单一结果表查询
      • Query－主从结果表，联动方式
      • Query－主从结果表，两表隐显方式
      • Query－统计图＋统计表方式
    2. CRUD、查询的新用例应该尽量采用这些模板，根据业务特点可作少量更改。
